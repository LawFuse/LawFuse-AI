# 类案推荐功能使用说明

## ✅ 已完成的功能

### 1. 后端部分

#### 文件结构
- `backfont/api/cases.js` - 类案推荐 API 接口
- `backfont/index.js` - 主服务器（已注册 cases 路由）
- `backfont/data/knowledge-index.json` - 知识库索引文件（2431个案例）

#### API 接口

**1. 类案推荐接口**
```
POST /api/cases/recommend
```

请求体：
```json
{
  "caseDescription": "案件描述文本"
}
```

响应：
```json
{
  "code": 0,
  "cases": [
    {
      "caseNumber": "(2016)京民申3726号",
      "category": "人格权纠纷+侵权",
      "similarity": 5,
      "preview": "案件摘要...",
      "fullContent": "完整案件内容..."
    }
  ]
}
```

**2. 重建知识库索引**
```
POST /api/cases/rebuild
```

用于在更新知识库文件后重新加载索引。

### 2. 前端部分

#### 修改的文件
- `vuefont/src/components/wenshujiexi.vue`

#### 功能流程

1. **用户输入案件描述**
2. **点击"类案推荐"按钮**
3. **系统执行以下步骤：**
   - 调用后端 `/api/cases/recommend` 接口
   - 从知识库检索最相关的 5 个案例
   - 将案例信息（案号、分类、内容）组装成 Prompt
   - 发送给 AI 生成类案推荐报告
   - AI 按照指定格式输出结果

#### AI 输出格式

```markdown
争议焦点:
（分析用户案件的核心争议点）

法律适用:
（列出适用的法律法规）

类案推荐

类案1：民事还是刑事/罪名/判案法院/判案时间/案件号/一审还是二审
案号：（确保案号准确无误）
案件名称：（根据案号和内容推断案件名称）
推荐理由：（说明与用户案件的相似点和参考价值）

类案2：...
类案3：...
类案4：...
类案5：...
```

## 🚀 使用方法

### 1. 启动后端服务

```bash
cd d:\makemoney1\48fal\backfont
npm start
```

服务将在 `http://localhost:3000` 运行

### 2. 启动前端服务

```bash
cd d:\makemoney1\48fal\vuefont
npm run dev
```

### 3. 使用类案推荐功能

1. 打开前端页面，进入"智能文书解析"模块
2. 在文本框中输入案件描述或粘贴文书内容
3. 点击"类案推荐"按钮
4. 等待系统返回类案推荐报告

## 🔍 技术特点

### 1. 检索算法
- 基于关键词匹配的相似度计算
- 支持中文分词
- 按相关度评分排序，返回 Top 5

### 2. 性能优化
- 知识库索引只需构建一次
- 索引文件加载到内存，检索速度 < 50ms
- 支持增量更新

### 3. 案号准确性
- 案号直接从知识库索引中获取
- 保证 100% 准确
- 避免 AI 幻觉问题

## 🔄 更新知识库

当添加新的案例文件到 `konwledge` 文件夹后：

```bash
# 在项目根目录运行
node kon.js
```

这会重新扫描所有文件并更新索引。

## 💡 注意事项

1. **案号准确性**: 系统从知识库直接获取案号，保证准确无误
2. **案件名称**: 由 AI 根据案号和案件内容合理推断
3. **推荐理由**: AI 会分析用户案件与检索案例的相似点
4. **Markdown 渲染**: 所有 AI 输出都经过 Markdown 格式化，显示美观

## 🐛 故障排查

### 问题1: 找不到相似案例
- 检查案件描述是否清晰
- 尝试补充更多案件细节
- 确认知识库中有相关类型的案例

### 问题2: 服务启动失败
- 确认已安装依赖: `npm install`
- 检查端口 3000 是否被占用
- 确认 `knowledge-index.json` 文件存在

### 问题3: 检索结果为空
- 运行 `node kon.js` 重新构建索引
- 检查 `konwledge` 文件夹路径是否正确
- 查看后端控制台日志

## ✨ 后续优化建议

1. **向量检索**: 可以升级为真正的向量数据库（如 ChromaDB + bge-m3 模型）
2. **语义搜索**: 提高检索准确率，不仅限于关键词匹配
3. **案例详情**: 添加点击查看完整案例的功能
4. **相似度阈值**: 设置最低相关度要求，过滤不相关的案例
5. **用户反馈**: 收集用户对推荐结果的评分，优化算法
